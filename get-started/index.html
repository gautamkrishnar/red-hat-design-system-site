Get started

Setting up

Our design kit is created in Adobe XD and includes various components and patterns for use in your designs. If the design kit is linked to your file, Adobe XD provides automatic updates of changes made to any of the components, so you’re always working with the latest designs. The design kit can also be viewed online if you need more design or development details about a component or pattern before you get started or while you’re working.

Download Adobe XD Browse the design kit online

Start working

In order to access our shared design kit via Adobe XD, please send us an email
(digital-systems-web-team@redhat.com) regarding your request to access the kit that must include the email associated with your Adobe account and the reason why you need access to the kit.

When you're ready to start working with the design system, follow these steps.

1

Get educated

Browse the website to familiarize yourself with our Foundations and Components.

2

Download the fonts

Download the Red Hat fonts so typography and components will render correctly in your designs.

3

Connect the design kit

In order to link the Adobe XD design kit to a new or existing file, follow these steps.

Show the Assets panel

  • Press Command + Shift + Y (macOS) or Control + Shift + Y (Windows)
  • Alternatively, click on the View toolbar menu and then Assets

Find the design kit cloud document

  • Under the Your Work title, click on Shared with You
  • If you have multiple cloud files, the design kit features a thumbnail of a grid
  • Click on the cloud file named Red-Hat-Digital-Design-System-Kit.xdc
  • The window will close and styles and components should be visible

If you can't see the cloud file or can't link the design kit to a file, contact us.

Connect the design kit

Link the Adobe XD design kit to a new or existing file to get access to the components and patterns.

4

Drag and drop

Start creating experiences by dragging and dropping components and patterns onto the design canvas.

Make a request

To request a new component or pattern, create an issue in our GitHub repo. To submit any changes to an existing component or pattern, contact us.

Call to action

Overview

Calls to action are styled links that direct users to other pages or sometimes display hidden content. They feature a text label, a background or a border, and icons.

Style

Calls to action can be used in light and dark themes. There are a variety of styles to choose from depending on what users need to accomplish, but some styles aren’t available in certain themes.

Theme

Primary

Use for the primary or the most important link. The white style should be used if the red style can’t be used (dark theme only). Don’t use this style multiple times in the same area.

Primary, video

Use to trigger an important video that will play in a Modal. Don't use this style without some kind of background underneath or use the Default, video style instead. This style can be used in both themes.

Secondary

Use for secondary or general links. This style is lower in hierarchy than the Primary style and it can be used multiple times in the same area.

Secondary, blue

Use instead of the Secondary style only if the design contains more blue than usual. Use in the light theme only.

Brick

Use to group a list of links together in a grid. Only Brick styles can stretch to fit a container or a grid, otherwise the text label padding stays the same in all other styles.

Bricks with icon

Use to group a list of links with icons together in a grid. Only Brick styles can stretch to fit a container or a grid, otherwise the text label padding stays the same in all other styles.

Default

Use for tertiary or the least important links. This style is the lowest in hierarchy and it can be used multiple times in the same area.

Default, video

Use to trigger a less important video that will play in a modal. This style is the lowest in hierarchy and it can be used multiple times in the same area.

Usage

A call to action is used to entice users to select a link or display hidden content, depending on the style. For example, use the Primary style to bring attention to an important link or use the Brick style to organize links in a grid that can also display hidden content.

Navigation vs. action

Use calls to action only as navigational elements and use Buttons to perform actions. Calls to action and buttons have stylistic differences, even though they look similar.

  • Calls to action are used more in layouts whereas buttons are used more in components.
  • The text labels in calls to action are larger and thicker than the text labels in buttons.
  • A call to action background container has more padding than a button background container.
  • A Primary call to action has a red background whereas a Primary button has a blue background.

Text labels

Call to action text labels are written differently than button text labels. Call to action text labels should be more creative or expressive to better entice users to make a selection.

Hierarchy

Calls to action are ordered by hierarchy from left to right or sometimes top to bottom. Similar styles can be used multiple times in the same area except for the Primary style.

Character count guidance

Calls to action should have limited character counts in order for the messaging to remain impactful. Avoid writing long calls to action because the text will break to two lines on smaller screens.

Style Maximum characters
Primary 30
Secondary 35
Brick 25
Default 45

Best practices

Don’t use more than three calls to action in a row because too many choices will make it hard for a user to decide which link to select.

Don’t use multiple Primary styles in the same area— that style is reserved to represent the one most important link.

Don’t stretch the other styles to fit a container or a grid. Only the Brick styles can stretch, otherwise the text label padding stays the same in all other styles.

Behavior

Call to action vs. Button

A call to action directs users to other pages or sometimes displays hidden content whereas a button performs an action. For example, a button shouldn’t be used as a link and a call to action shouldn’t be used to submit a form. Calls to action and buttons can be used in the same area as long as the styles are used properly and perform the correct functions.

Bricks

Sometimes the Brick styles can act like an Accordion where they slide vertically to reveal hidden content when triggered. Otherwise, they act like a list of links that are grouped together in a grid.

Responsive design

Calls to action maintain the same scale on all screen sizes. If they stack on mobile, the alignment and the order of hierarchy stays the same, which is left to right and top to bottom. If a call to action contains a background or a border, the text label stays centered if it’s really long and if it breaks to two lines.

Call to Action - NJK

Overview

A Call to action is a link that directs users to other pages or displays extra content when selected. Call to action styles feature a text label, a background or a border, and icons.

Style

Calls to action can be used in light and dark themes. There are a variety of styles to choose from depending on what users are trying to do, but some styles aren’t available in certain themes.

Primary

Use for the primary or the most important link. The white style should be used if the red style can’t be used in the dark theme only. Don’t use this style multiple times in the same area.

Primary, video

Use to trigger an important video that will play in a Modal . Don’t use this style without a background underneath or use the Default, video style instead. This style can be used in both themes.

Secondary

Use for secondary or general links. This style is lower in hierarchy than the Primary style and it can be used multiple times in the same area.

Secondary, blue

Use instead of the Secondary style only if the design contains more blue than usual. Use in the light theme only.

Brick

Use to group a list of links together in a grid. Only Brick styles can stretch to fit a container or a grid, otherwise the label padding stays the same in all other styles.

Bricks with icon

Use to group a list of links with icons together in a grid. Only Brick styles can stretch to fit a container or a grid, otherwise the label padding stays the same in all other styles.

Default

Use for tertiary or the least important links. This style is the lowest in hierarchy and it can be used multiple times in the same area.

Default, video

Use to trigger a less important video that will play in a modal. This style is the lowest in hierarchy and it can be used multiple times in the same area.

Usage

A Call to action is used to entice users to select a link or display hidden content, depending on the style. For example, use the Primary style to bring attention to an important link or use the Brick style to organize links in a grid that also display hidden content.

Navigation vs. action

Use Calls to action only as navigational elements and use Buttons to perform actions.

Labels

Call to action labels are written differently than Button labels. Call to action labels should be more creative or expressive to better entice users to make a selection.

Hierarchy

Calls to action are ordered by hierarchy from left to right or sometimes top to bottom (. Similar styles can be used multiple times in the same area except for the Primary and Primary, disabled styles.

Best practices

Don’t use more than three Calls to action in a row because too many choices will make it hard for users to decide which link to select.

Don’t use multiple Primary styles in the same area, that style is reserved to represent the one most important link.

Don’t stretch the other styles to fit a container or a grid. Only the Brick styles can stretch, otherwise the label padding stays the same in all other styles.

Behavior

Call to action vs. Button

A Call to action directs users to other pages or sometimes displays hidden content whereas a Button performs an action. For example, a Button shouldn’t be used as a link and a Call to action shouldn’t be used to submit a form. Calls to action and Buttons can be used in the same area as long as the styles are used properly and perform the right functions.

Responsive design

Calls to action maintain the same scale on desktop and mobile. If they stack, the alignment and the order of hierarchy stays the same, which is left to right and top to bottom. If a Call to action contains a background or a border, the label stays centered if it’s really long and if it breaks to two lines.

Search bar

Overview

A Search bar is a grouping of a form field and a search button. It’s a modular component that can be used in a variety of layouts.

Sample component

Component status

Style

Search bars can be used in light and dark themes. They consist of a form field with placeholder text inside and a search button that’s placed on the right.

Theme

Usage

A search bar allows users to search for something by entering text in the form field and then clicking on or tapping the search button. Search results are displayed on another page or below the search bar.

Layout

Search bars can be used in large and small layouts. They have a minimum width, but no maximum width other than the boundaries of the container that it’s placed in or the grid.

Placeholder text

Search bars contain placeholder text that indicates how broad or specific the search results will be. For example, placeholder text might state Search all Red Hat if the search results aren’t specific to one page.

Generic placeholder text will yield more broad search results

More specific placeholder text will yield more tailored search results

Best practices

Don’t replace the search button with a call to action.

Don’t use a different button style, even if the text is the same.

Don't use the search button by itself.

Don't place the search button below the form field or change its aspect ratio, even on small screens.

Behavior

Form field

When you click or tap inside the form field, the placeholder text and its styling will remain the same. When a user starts typing new text, the styling changes.

Typeahead

In some search bars, when a user begins to type their search term, a pattern appears below the form field that gives suggestions as to what a user might be searching for.

Responsive design

Search bars can stretch horizontally to fit any container, but only the form field stretches whereas the button always stays the same size.

Interaction states

Both the form field and the search button have interaction states.

Spacing

Search bars use PatternFly 4 spacers to define spacing values between elements.

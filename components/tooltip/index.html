Tooltip

Overview

A Tooltip is a user-triggered floating text area that provides helpful or contextual information.

Sample component

Component status

Style

A tooltip can be used with a light or dark background. It consists of plain text wrapped in a background container with rounded corners and a subtle drop shadow (dark tooltip only). The background container also includes an arrow that points to a Help or Information icon.

Theme

Usage

Content

Content in a tooltip is limited to text only and should be brief and helpful to a user. If content needs to be more thorough or interactive elements need to be included, use a Modal or even a Disclosure instead.

Icon pairing

A tooltip should always be paired with a small help or information icon, it shouldn't appear over any links or components.

Orientations

Depending on the amount of content or available screen space, a tooltip can be used in different orientations. There’s no default, but when choosing an orientation, be mindful of how a tooltip can potentially cover up important content or get cut off by the browser window.

Best practices

For better accessibility, don’t use a light tooltip in a light environment because it will blend into the background too much.

Content inside of a tooltip shouldn’t be cut off by the browser window, change the tooltip orientation if it does.

Don’t pair a tooltip with any links or components.

Behavior

How a user interacts with a tooltip depends on what the screen size is.

  • On large screens, a tooltip disappears after a user moves their cursor off of the icon
  • On large screens, a tooltip disappears after the focus indicator is moved away from the icon
  • On small screens, a tooltip disappears after a user taps the icon again or taps outside of the tooltip

Responsive design

A tooltip can work on both large and small screens, depending on the amount of content or available screen space.

Breakpoints

When a tooltip is used on smaller screens, its orientation might change depending on how content is arranged.

Interaction states

The only interaction state is a tooltip appearing next to an icon when focused, hovered, or tapped.

Spacing

A tooltip uses PatternFly 4 spacers to define spacing values between elements. Every tooltip orientation contains the same amount of container padding and exterior spacing in between the arrow and icon.

Alert

Overview

Alerts are banners used to notify a user about a change in status or communicate other information. They can appear with or without a user triggering a specific action.

Sample component

Component status

Style

Alerts can be used in the light theme only. There are two alert variants that feature multiple styles, depending on the severity of the message that's being communicated.

Inline

Inline alerts can be placed in a container or a component and shouldn’t block other content. The required elements include a bar or a border, an icon, a title, and a container. Text that provides more information about the alert may or may not be included. Sometimes a close button is required if an inline alert needs to be dismissible.

The alternate inline alert style features a border instead of a horizontal line on top which is used to express more urgency or to grab the attention of a user.

Toast

Toast alerts slide into view in the top right corner of the page and rest on top of content. The required elements include a bar, an icon, a title, a close button, and a white container with a drop shadow. Text that provides more information about the alert may or may not be included.

Usage

Alerts are notifications that appear or are generated to communicate with users or attract their attention.

Inline

Use to present simple information or a local notification, like after a user submits a form with errors. Inline alerts should be informational or identify any actions that are required before a user can continue. They should also have the option to be dismissible.

Toast

Use to present a notification about an important global event or a change. Toast alerts shouldn’t interrupt a user’s workflow therefore they should always be dismissible.

Severity

Severity communicates the urgency of an alert. The combination of a status icon, a title, and the container style all help to clarify severity levels of alerts.

Default

Use to indicate generic information or a message with no severity.

Info

Use to indicate helpful information or a message with little to no severity.

Success

Use to indicate success information, like if a task was completed successfully.

Warning

Use to indicate a cautious severity, like a non-blocking error.

Danger

Use to indicate a critical severity, like a blocking error.

Positioning

Inline

Inline alerts appear at the top of a container or a component and push other content down. Inline alerts also have no fixed width, they conform to the width of whatever container they’re placed in and they persist until a user dismisses them or navigates away from the page.

Toast

Toast alerts slide in from the top right edge of the screen under the navigation and overlay content. Toast alerts also have a fixed width of 600px and they persist until they time out after eight seconds or when a user dismisses them.

Best practices

Don't use inline alerts to communicate messages about an important global event or a change.

Don’t use Toast alerts to present simple information or a local notification.

Don’t use different alert variants in the same area.

Behavior

Stacking

If the same alert variant appears in the same area, they stack. The most recent alert should appear at the top of the stack and push other alerts down. When stacked alerts are dismissed or time out, the remaining alerts will move up and fill the empty space.

Responsive design

On small screens, both alert variants will span the full column and be positioned in the center of the layout. Toast alerts will continue to stack on top of each other, so be mindful of how much vertical space the group will occupy if there are multiple alerts.

Breakpoints

Alerts are banners that respond to changes in screen size just like other layouts.

Large screens

The maximum width of a Toast alert on all screens is six grid columns

Small screens

Form

Overview

A form is a group of elements that are used to collect information from users. It features a combination of text labels, fields, data inputs, and buttons.

Sample component

Component status

Style

There are a variety of elements that can be used in a form depending on the kind of information that needs to be collected from users.

Text labels

Text labels tell a user what information they need to enter in each field or input.

Fields

Fields provide spaces for a user to enter text.

Data inputs

Data inputs provide additional ways for a user to submit information that isn’t text.

Buttons

Buttons enable a user to submit or exit a form.

Usage

Forms guide users through a task while gathering information from them.

Standard form

A standard form is positioned in a card that usually spans four columns.

Dynamic form

A dynamic form can be used without a card. It spans a maximum of eight columns and it can be positioned in different parts of a page layout.

Content

Form content is how you speak to users. They can submit a form faster if all elements are clearly labeled.

Field text labels

A field text label can be used for critical information, like a user’s name or phone number.

Form field

Placeholder text can be used in a field as an example of required or recommended text.

Helper text

Helper text can be used to provide clarity about what exactly a user needs to input.

Tooltips

Tooltips can be used to provide additional context or helpful information.

Required fields

A required field features an asterisk (*) to the right of the field text label.

If all fields are required, don’t use an asterisk above every field. Instead, put a message at the top of the form stating All fields are required. If all fields are optional, the message should state All fields are optional.

Data inputs

Data inputs provide additional ways for users to submit information that isn’t text. Each data input serves a different purpose and they can all be used in the same area.

Arranging data inputs

Radio buttons and checkboxes can be arranged horizontally or vertically depending on how much space there is.

  • If maintaining vertical space is important and there are fewer than three options, arrange data inputs horizontally
  • If there are more than three options, arrange data inputs vertically
  • If there are no space constraints and the text labels are short enough, arrange data inputs in multiple columns, but only if there are three or more options

Placement

A form can be used on different kinds of pages, but placed away from blocks of content if there’s enough space. The width of a form should reflect the amount of content that needs to be collected, but it should never exceed 750px to maintain optimal readability. For example, if a user is expected to submit lots of text, the form should be wider.

Best practices

Buttons should always be placed on the bottom left and be aligned to the left edge of fields. They should also be arranged by hierarchy from left to right.

Don't use multiple Primary buttons in a form.

Behavior

Errors

It's important that error state messages explain to users how they can fix the errors beyond only stating that there are errors.

Errors are displayed after an incomplete field or data input loses focus

Errors are displayed when a user submits a form without correcting any errors first

Responsive design

If a form is wide enough, field text labels can be placed to the left of fields. When a form scales down on smaller screens, field text labels move above fields and are aligned to the left edge.

Interaction states

All elements within a form have interaction states except if they’re disabled.

Spacing

A form uses PatternFly 4 spacers to define spacing values between elements.
